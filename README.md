# Вертикаль — Клиентское мобильное приложение

> Мобильное приложение для записи на тренировки в скалодроме «Вертикаль»

---

## О проекте

Клиентское мобильное приложение для скалодрома «Вертикаль», которое позволяет клиентам:
- Просматривать расписание тренировок на 7 дней вперед
- Записываться на групповые тренировки с выбором формата (болдеринг / трассы с веревкой)
- Выбирать опцию проката снаряжения (скальники, страховочная система)
- Отменять запись (с ограничениями по времени)
- Оценивать инструкторов после тренировки
- Получать push-уведомления о напоминаниях и изменениях в расписании

**Текущий скоуп поставки:** только клиентская часть (мобильное приложение + API для него).
Интерфейсы инструктора и администратора уже реализованы в существующей инфраструктуре.

---

## Ключевые бизнес-требования

| Требование | Описание |
|------------|----------|
| **Просмотр расписания** | Клиент видит слоты на ближайшие 7 дней (горизонт планирования) |
| **Запись на тренировку** | Выбор формата, инструктора, времени, опции проката снаряжения |
| **Ограничение по количеству участников** | До 16 человек в группе (до 8 — для новичковых тренировок по безопасности) |
| **Отмена записи** | Отмена доступна, но за 10 минут до начала — проблемная зона (требуется дополнительное обсуждение) |
| **Оценка инструктора** | После тренировки клиент может поставить оценку |
| **Уведомления** | Push-напоминания о предстоящих тренировках и отменах со стороны скалодрома |
| **Статус «Отменена скалодромом»** | При профилактике или отмене организатором бронь сохраняется со специальным статусом и причиной |

## 🏗️ Архитектура и ограничения

### Важные архитектурные решения

- **Backend как black box**: существующая инфраструктура — источник истины.
  - Приложение полагается на ответы API
  - Гарантия отсутствия двойных броней — на стороне бэкенда
  - Клиентское приложение корректно обрабатывает отказ при отсутствии мест

- **Только клиентский интерфейс**:
  - Интерфейсы инструктора и администратора уже существуют
  - В текущей поставке не разрабатываются

- **Данные**: нет исторических данных и легаси-схем
  - Каноническая схема — контракт API
  - Все поля контракта считаются доступными (прокатный тариф, зона/формат, свободный фонд снаряжения)

## 📱 Основные экраны (User Stories)

### MVP (первая версия)

1. **Экран расписания**
   - Список доступных слотов на 7 дней
   - Карточка слота: время, формат, инструктор, свободные места
   - Фильтр по датам (для более длительного периода)
   - Empty State: «Пока нет доступных тренировок»

2. **Экран деталей слота**
   - Подробная информация о тренировке
   - Выбор опции проката снаряжения
   - Кнопка «Записаться»

3. **Экран моих записей**
   - Список будущих и прошедших тренировок
   - Статусы: «Подтверждена», «Отменена скалодромом» (с причиной)
   - Возможность отменить запись (если еще не поздно)

4. **Экран оценки инструктора**
   - Появляется после завершения тренировки
   - Возможность поставить оценку

5. **Экран уведомлений**
   - История push-уведомлений
   - Напоминания о предстоящих тренировках
   - Уведомления об отмене скалодромом
