# Карта маршрута (навигация)

> **ID:** LOGIC-006
> **Тип:** Навигационная схема
> **Приоритет:** P0
> **Связанные требования:** Все экраны; UC-015; NFR-020, NFR-028, NFR-029

---

## 1. Назначение

Документация навигационной структуры приложения, описывающая все экраны и переходы между ними, включая глубокие ссылки из push-уведомлений и обработку навигации в разных состояниях приложения (закрыто, фон, активно).

**Сценарии:**
- Навигация между основными экранами
- Глубокие ссылки из push-уведомлений
- Обработка навигации при разных состояниях приложения
- Восстановление состояния при перезапуске

## 3. Матрица переходов

### 3.1. С главного экрана

| Целевой экран | Действие | Тип перехода |
|---------------|----------|--------------|
| Расписание | Кнопка "Всё расписание" | Push |
| Мои записи | Кнопка "Мои записи" | Push |
| Абонемент | Кнопка "Абонемент" | Push |
| Профиль | Нажатие на аватар | Push |
| Детали слота | Нажатие на карточку слота | Push |
| Детали записи | Нажатие на карточку "Следующая тренировка" | Push |

### 3.2. С экрана расписания

| Целевой экран | Действие | Тип перехода |
|---------------|----------|--------------|

| Главный экран | Кнопка "Назад" | Pop |

### 3.3. С экрана деталей слота

## 4. Глубокие ссылки (Deep Links)

### 4.1. Схемы глубоких ссылок

| Схема | Формат | Пример |
|-------|--------|--------|
| **Схема приложения** | `vertical://{path}` | `vertical://bookings` |
| **Универсальная ссылка** | `https://vertical-climbing.ru/{path}` | `https://vertical-climbing.ru/slot/slot-001` |

### 4.2. Поддерживаемые пути

| Путь | Описание | Целевой экран | Параметры |
|------|----------|---------------|-----------|
| `/slot/{slotId}` | Открыть детали слота | SCR-004 | slotId |
| `/booking/{bookingId}` | Открыть детали записи | SCR-006 | bookingId |
| `/bookings` | Открыть мои записи | SCR-005 | — |
| `/profile` | Открыть профиль | SCR-007 | — |
| `/subscription` | Открыть абонемент | SCR-008 | — |
| `/rating/{instructorId}` | Открыть оценку инструктора | SCR-009 | instructorId |

### 4.3. Обработка глубоких ссылок

| Состояние приложения | Действие |
|----------------------|----------|
| **Закрыто** | Открыть приложение, выполнить навигацию к целевому экрану |
| **Фоновое** | Восстановить приложение, выполнить навигацию к целевому экрану |
| **Активное** | Выполнить навигацию к целевому экрану (push) |

## 6. Обработка навигации

### 6.1. Типы переходов

| Тип | Описание | Использование |
|-----|----------|---------------|
| **Push** | Добавление экрана в стек навигации | Основной переход между экранами |
| **Pop** | Удаление текущего экрана из стека | Возврат назад |
| **Pop to root** | Возврат к корневому экрану стека | После успешной записи |
| **Clear stack** | Очистка всей истории навигации | Выход из аккаунта |

### 6.2. Глубина стека

| Ограничение | Значение |
|-------------|----------|
| **Максимальная глубина** | 5 экранов |
| **Восстановление** | При перезапуске сохраняется последний экран |
| **Очистка** | При выходе из аккаунта стек полностью очищается |

### 6.3. Анимация переходов

| Тип перехода | Анимация | Длительность |
|--------------|----------|--------------|
| Push | Slide справа налево | 0.3 сек |
| Pop | Slide слева направо | 0.3 сек |
| Pop to root | Fade | 0.2 сек |
| Модалка | Slide снизу вверх | 0.3 сек |
| Закрытие модалки | Slide сверху вниз | 0.3 сек |

---

## 7. Критерии приёмки

1. Все переходы между экранами работают согласно матрице переходов
2. Глубокие ссылки корректно открывают целевые экраны
3. Push-уведомления открывают правильные экраны при нажатии
4. При закрытом приложении push-уведомление открывает приложение и целевой экран
5. При фоновом приложении push-уведомление восстанавливает приложение и открывает целевой экран
6. При активном приложении push-уведомление выполняет навигацию без перезапуска
7. Кнопка "Назад" работает корректно на всех экранах
8. Выход из аккаунта очищает стек навигации
9. Анимации переходов плавные и соответствуют дизайн-системе
10. Глубина стека не превышает 5 экранов
