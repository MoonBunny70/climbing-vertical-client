# Запрос push-разрешения

> **ID:** LOGIC-007
> **Тип:** Бизнес-логика / Сценарий
> **Приоритет:** P0
> **Связанные требования:** NFR-016, NFR-027; UC-013; US-080, US-081, US-082

---

## 1. Назначение

Логика запроса разрешения на отправку push-уведомлений в операционной системе Android. Обеспечивает получение FCM-токена для отправки уведомлений и корректную обработку всех сценариев отказа.

**Сценарии:**
- Запрос разрешения при первом запуске
- Повторный запрос при отказе
- Обработка разрешения в настройках ОС
- Получение и регистрация FCM-токена
- Обработка ошибок получения токена

## 3. Контекстный запрос разрешения

### 3.1. Моменты запроса

| Сценарий | Момент запроса | Обоснование |
|----------|----------------|-------------|
| **Первый запуск** | После входа в приложение | Пользователь уже заинтересован в использовании |
| **Повторный запрос** | При первом действии, требующем уведомлений | Не раздражать пользователя повторными запросами |
| **После регистрации** | Сразу после создания аккаунта | Пользователь уже в системе, готов получать уведомления |

### 3.2. Текст объяснения

| Сценарий | Заголовок | Описание |
|----------|-----------|----------|
| **Первый запрос** | "Получать уведомления?" | "Мы будем присылать напоминания о тренировках, сообщать об отменах и изменениях. Вы сможете настроить время напоминаний в профиле." |
| **Повторный запрос** | "Включить уведомления?" | "Без уведомлений вы можете пропустить важные изменения в расписании. Включить уведомления в настройках?" |

---

## 4. Состояния разрешения

| Состояние | Описание | Действие |
|-----------|----------|----------|
| **Не запрошено** | Разрешение ещё не запрашивалось | Запросить при первом удобном случае |
| **Запрошено** | Запрос отправлен, ожидание ответа | Ожидать ответа пользователя |
| **Предоставлено** | Пользователь дал разрешение | Получить FCM-токен, зарегистрировать на сервере |
| **Отказано** | Пользователь отказал | Больше не запрашивать автоматически |
| **Отключено в настройках** | Пользователь отключил в настройках ОС | Показывать баннер с предложением включить |

### 5.2. Обработка ошибок получения токена

| Ошибка | Причина | Действие |
|--------|---------|----------|
| `SERVICE_NOT_AVAILABLE` | Google Play Services недоступны | Повторная попытка через 30 секунд |
| `NETWORK_ERROR` | Отсутствует интернет | Повторная попытка через 60 секунд |
| `TOKEN_REFRESH` | Токен обновлён | Регистрация нового токена на сервере |

## 6. Регистрация токена на сервере

### 6.1. Эндпоинт регистрации

| Элемент | Описание |
|---------|----------|
| **Метод** | POST |
| **Путь** | `/notifications/register` |
| **Тело** | `{ fcmToken: string }` |
| **Авторизация** | Bearer token (JWT) |

### 6.2. Ответы сервера

| Код | Значение | Действие |
|-----|----------|----------|
| 200 | Успешная регистрация | Уведомления готовы к отправке |
| 400 | Неверный формат токена | Повторная попытка с новым токеном |
| 401 | Невалидный токен авторизации | Обновить токен, повторить запрос |
| 503 | Сервер недоступен | Повторная попытка с экспоненциальной задержкой |

---

## 7. Повторный запрос при отказе

### 7.1. Стратегия повторных запросов

| Попытка | Момент | Способ |
|---------|--------|--------|
| 1 | Первый запуск | Системный диалог |
| 2 | При первой записи | Контекстный диалог с объяснением |
| 3 | При переходе в настройки | Баннер "Включите уведомления" |

### 7.2. Информирование о преимуществах

| Преимущество | Описание |
|--------------|----------|
| **Напоминания** | Не пропустите тренировку, получите напоминание за выбранное время |
| **Отмены** | Узнайте об отмене скалодромом и запишитесь на другой слот |
| **Изменения** | Будьте в курсе изменений времени или инструктора |
| **Оценка** | Получите напоминание об оценке инструктора |

## 8. Проверка статуса в настройках ОС

### 8.1. Периодическая проверка

| Элемент | Описание |
|---------|----------|
| **Частота** | При каждом запуске приложения |
| **Метод** | Проверка через NotificationManagerCompat |
| **Действие при включении** | Получить и зарегистрировать FCM-токен |
| **Действие при отключении** | Отправить запрос на удаление токена |

### 8.2. Отображение статуса

| Статус в ОС | Статус в приложении | Отображение |
|-------------|---------------------|-------------|
| Включено | Активно | "🔔 Уведомления включены" |
| Включено | Не зарегистрировано | "Идёт регистрация..." |
| Отключено | Отключено | "🔕 Уведомления отключены в настройках системы" |

---

## 9. Удаление токена при выходе

### 9.1. Поток удаления

| Событие | Действие |
|---------|----------|
| Выход из аккаунта | Отправить запрос на удаление FCM-токена |
| Очистка данных приложения | Удалить локально сохранённый токен |
| Смена аккаунта | Удалить старый токен, зарегистрировать новый |

### 9.2. Эндпоинт удаления

| Элемент | Описание |
|---------|----------|
| **Метод** | DELETE |
| **Путь** | `/notifications/unregister` |
| **Тело** | `{ fcmToken: string }` |
| **Авторизация** | Bearer token (JWT) |

---

## 10. Критерии приёмки

1. Запрос разрешения показывается после успешного входа в приложение
2. Запрос сопровождается понятным объяснением на русском языке
3. При предоставлении разрешения FCM-токен успешно получается
4. Токен успешно регистрируется на сервере
5. При отказе разрешения запрос повторяется при первом действии с уведомлениями
6. При повторном отказе больше не запрашивать автоматически
7. При изменении статуса в настройках ОС приложение корректно обновляет статус
8. При выходе из аккаунта токен удаляется с сервера
9. Ошибки получения токена обрабатываются с повторными попытками
10. Все сообщения пользователю понятны и на русском языке
