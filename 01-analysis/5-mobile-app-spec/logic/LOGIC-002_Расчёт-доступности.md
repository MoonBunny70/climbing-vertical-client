# Расчёт доступности слотов

> **ID:** LOGIC-002
> **Тип:** Бизнес-логика / Алгоритм
> **Приоритет:** P0
> **Связанные требования:** FR-013, FR-014, FR-015; UC-003; US-010

---

## 1. Назначение

Логика определения статуса и доступности слота для отображения в карточке расписания. На основе данных от API определяет, доступен ли слот для записи, сколько мест свободно, и какой визуальный статус присвоить карточке.

**Сценарии:**
- Определение доступности слота для записи
- Расчет цветового индикатора свободных мест
- Отображение статуса "Отменён скалодромом"
- Расчет процента заполненности мест

---

## 2. Исходные данные из API

Слот возвращает следующие поля, необходимые для расчёта:

```json
{
  "id": "slot-001",
  "totalSpots": 8,
  "availableSpots": 5,
  "status": "AVAILABLE",
  "cancellationReason": null
}
```

## Маппинг статусов для UI

| Статус API | availableSpots | Статус для UI | Цвет индикатора | Отображаемый текст |
|------|----------|----------|----------|----------|
| AVAILABLE | > 3 | AVAILABLE | 🟢 Зелёный | "Осталось X мест из Y" |
| AVAILABLE | 1–3 | LOW_AVAILABILITY | 🟡 Жёлтый | "Осталось X мест из Y" |
| AVAILABLE | 0 | FULL | 🔴 Красный | "Мест нет" |
| CANCELLED_BY_GYM | 0 | CANCELLED | ⚪ Серый | "ОТМЕНЕН СКАЛОДРОМОМ" |

### 4.1 Детализация по состоянию

Состояние AVAILABLE (много мест)

✅ Слот доступен для записи
- availableSpots > 3
- Индикатор: Зелёный (#22C55E)
- Текст: "Осталось {availableSpots} мест из {totalSpots}"
- Кнопка: Активна

Состояние LOW_AVAILABILITY (мало мест)

⚠️ Слот доступен, но мест мало
- availableSpots от 1 до 3
- Индикатор: Жёлтый (#F59E0B)
- Текст: "Осталось {availableSpots} места из {totalSpots}" (склонение по правилам русского языка)
- Кнопка: Активна

Состояние FULL (мест нет)

❌ Слот недоступен
- availableSpots = 0
- Индикатор: Красный (#EF4444)
- Текст: "Мест нет"
- Кнопка: Неактивна

Состояние CANCELLED (отменён скалодромом)

⚪ Слот отменён
- status = "CANCELLED_BY_GYM"
- Индикатор: Серый (#9CA3AF)
- Текст: "ОТМЕНЕН СКАЛОДРОМОМ"
- Доп. текст: причина отмены (если есть)
- Кнопка: Неактивна

## 5. Склонение числительных

fun getSpotsText(count: Int, total: Int): String {
    val spotsWord = when {
        count % 10 == 1 && count % 100 != 11 -> "место"
        count % 10 in 2..4 && count % 100 !in 12..14 -> "места"
        else -> "мест"
    }
    return "Осталось $count $spotsWord из $total"
}

## 6. Расчёт процента заполненности

Для визуализации прогресс-бара в карточке слота или на экране деталей.

### 6.1 Формула

filledPercentage = ((totalSpots - availableSpots) / totalSpots) * 100

### 6.2

| Заполненность | Цвет | HEX |
|------|----------|----------|
| 0–50% | > 3 | Зелёный | #22C55E |
| 51–85% | 1–3 | Жёлтый | #F59E0B |
| 86–100% | 0 | Красный | #EF4444 |

## 7. Формула для unavailable-слотов

Слот недоступен для записи в следующих случаях:

| Заполненность | Цвет |
|------|----------|
| status == "CANCELLED_BY_GYM" | Слот отменён скалодромом |
| availableSpots == 0 | Все места заняты |
| startTime < currentTime | Тренировка уже началась |

Кнопка "Записаться" должна быть неактивной во всех трёх случаях.
