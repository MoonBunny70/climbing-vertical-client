# Обработка push-уведомлений

> **ID:** LOGIC-010
> **Тип:** Бизнес-логика / Сценарий
> **Приоритет:** P0
> **Связанные требования:** FR-080, FR-081, FR-082, FR-083, FR-084, FR-085; UC-008, UC-009, UC-010, UC-013, UC-015; US-080, US-081, US-082, US-083; NFR-027, NFR-028, NFR-029

---

## 1. Назначение

Логика обработки входящих push-уведомлений, включая их получение, отображение, обработку нажатий и навигацию по глубоким ссылкам. Обеспечивает корректную доставку уведомлений в разных состояниях приложения и предоставляет пользователю возможность быстрого действия.

**Сценарии:**
- Получение push-уведомления в активном состоянии
- Получение push-уведомления в фоновом состоянии
- Получение push-уведомления при закрытом приложении
- Обработка нажатия на уведомление с глубокой ссылкой
- Обновление UI при получении уведомления

---

## 2. Типы push-уведомлений

### 2.1. Перечень типов

| Тип | Описание | Когда отправляется | Действие по нажатию |
|-----|----------|-------------------|---------------------|
| **REMINDER** | Напоминание о предстоящей тренировке | За `n` минут до начала (настраивается клиентом) | Открыть детали записи |
| **CANCELLED_BY_GYM** | Отмена тренировки скалодромом | При отмене слота администратором | Открыть расписание для записи на другой слот |
| **TIME_CHANGED** | Изменение времени начала тренировки | При сдвиге времени слота | Открыть детали записи |
| **INSTRUCTOR_CHANGED** | Смена инструктора на тренировке | При замене инструктора | Открыть детали записи |
| **RATING_REMINDER** | Напоминание об оценке инструктора | Через 1 час после окончания тренировки | Открыть форму оценки инструктора |

### 2.2. Приоритеты уведомлений

| Приоритет | Типы уведомлений | Поведение |
|-----------|------------------|-----------|
| **Высокий** | CANCELLED_BY_GYM | Показывать сразу, звук, вибрация |
| **Средний** | TIME_CHANGED, INSTRUCTOR_CHANGED | Показывать сразу, звук |
| **Низкий** | REMINDER, RATING_REMINDER | Показывать в тихом режиме |

## 3. Структура push-уведомления

### 3.1. Обязательные поля

| Поле | Тип | Описание | Пример |
|------|-----|----------|--------|
| `title` | string | Заголовок уведомления | "Напоминание о тренировке" |
| `body` | string | Текст уведомления | "У вас тренировка завтра в 18:00" |
| `type` | string | Тип уведомления | "REMINDER" |
| `data` | object | Дополнительные данные для обработки | См. таблицу ниже |

### 3.2. Поля в data в зависимости от типа

| Тип | Поля в data | Описание |
|-----|-------------|----------|
| **REMINDER** | `bookingId` | ID записи |
| | `slotId` | ID слота |
| | `startTime` | Время начала тренировки |
| **CANCELLED_BY_GYM** | `slotId` | ID отменённого слота |
| | `reason` | Причина отмены |
| | `bookingId` | ID записи (если была) |
| **TIME_CHANGED** | `bookingId` | ID записи |
| | `oldStartTime` | Старое время начала |
| | `newStartTime` | Новое время начала |
| **INSTRUCTOR_CHANGED** | `bookingId` | ID записи |
| | `oldInstructorName` | Старое имя инструктора |
| | `newInstructorName` | Новое имя инструктора |
| **RATING_REMINDER** | `instructorId` | ID инструктора |
| | `instructorName` | Имя инструктора |
| | `bookingId` | ID записи (опционально) |

## 4. Обработка в разных состояниях приложения

### 4.1. Матрица состояний

| Состояние приложения | Отображение уведомления | Обработка данных |
|----------------------|------------------------|------------------|
| **Активное (Foreground)** | Кастомный in-app баннер | Сохранение данных, обновление UI |
| **Фоновое (Background)** | Системное уведомление | Сохранение данных при открытии |
| **Закрытое (Terminated)** | Системное уведомление | Обработка при запуске |

### 4.2. Активное состояние (Foreground)

| Шаг | Действие |
|-----|----------|
| 1 | Получение push-уведомления |
| 2 | Отображение in-app баннера в верхней части экрана |
| 3 | Сохранение данных уведомления в локальном хранилище |
| 4 | Обновление UI (если необходимо) |
| 5 | При нажатии на баннер — выполнение глубокой ссылки |

### 4.3. Фоновое состояние (Background)

| Шаг | Действие |
|-----|----------|
| 1 | Получение push-уведомления |
| 2 | Отображение системного уведомления |
| 3 | Сохранение данных уведомления в локальном хранилище |
| 4 | При нажатии на уведомление — открытие приложения и выполнение глубокой ссылки |

### 4.4. Закрытое состояние (Terminated)

| Шаг | Действие |
|-----|----------|
| 1 | Получение push-уведомления |
| 2 | Отображение системного уведомления |
| 3 | При нажатии на уведомление — запуск приложения |
| 4 | Обработка данных из уведомления |
| 5 | Выполнение глубокой ссылки |

## 5. In-app баннер (активное состояние)

### 5.1. Визуальное отображение

┌─────────────────────────────────────────────────────────────────┐
│ 🔔 Напоминание о тренировке │ ← Заголовок
│ У вас тренировка завтра в 18:00. Не забудьте прийти! │ ← Текст
│ [Открыть] │ ← Кнопка действия
└─────────────────────────────────────────────────────────────────┘


### 5.2. Поведение баннера

| Свойство | Описание |
|----------|----------|
| **Появление** | Slide сверху вниз (0.3 сек) |
| **Исчезновение** | Slide снизу вверх (0.3 сек) или по таймеру (5 сек) |
| **Нажатие** | Выполнение глубокой ссылки |
| **Свайп вверх** | Закрытие баннера |
| **Автозакрытие** | Через 5 секунд |

## 6. Глубокие ссылки из push

### 6.1. Маппинг типов на глубокие ссылки

| Тип уведомления | Глубокая ссылка | Параметры |
|-----------------|-----------------|-----------|
| **REMINDER** | `vertical://booking/{bookingId}` | bookingId |
| **CANCELLED_BY_GYM** | `vertical://slots` | — |
| **TIME_CHANGED** | `vertical://booking/{bookingId}` | bookingId |
| **INSTRUCTOR_CHANGED** | `vertical://booking/{bookingId}` | bookingId |
| **RATING_REMINDER** | `vertical://rating/{instructorId}` | instructorId |

### 6.2. Обработка глубокой ссылки

| Шаг | Действие |
|-----|----------|
| 1 | Извлечение пути и параметров из ссылки |
| 2 | Проверка авторизации пользователя |
| 3 | Навигация к целевому экрану |
| 4 | Если необходимо — загрузка данных для экрана |

## 7. Обновление UI при получении push

### 7.1. Типы обновлений

| Тип уведомления | Обновляемый экран | Действие |
|-----------------|-------------------|----------|
| **REMINDER** | — | Без обновления (только уведомление) |
| **CANCELLED_BY_GYM** | Расписание, Мои записи | Обновить списки, инвалидировать кеш |
| **TIME_CHANGED** | Мои записи, Детали записи | Обновить карточку записи |
| **INSTRUCTOR_CHANGED** | Мои записи, Детали записи | Обновить имя инструктора |

### 7.2. Инвалидация кеша при получении

| Тип уведомления | Инвалидируемые данные |
|-----------------|----------------------|
| **CANCELLED_BY_GYM** | Расписание, Мои записи |
| **TIME_CHANGED** | Мои записи, Расписание |
| **INSTRUCTOR_CHANGED** | Мои записи, Расписание |
| **REMINDER** | — |
| **RATING_REMINDER** | — |

## 8. Обработка ошибок

### 8.1. Ошибки при получении push

| Ошибка | Действие |
|--------|----------|
| **FCM-токен не зарегистрирован** | Повторная попытка регистрации |
| **Ошибка обработки данных** | Логирование ошибки, показ уведомления без глубокой ссылки |
| **Пользователь не авторизован** | Сохранение уведомления, обработка после входа |
| **Целевой экран недоступен** | Переход на главный экран |

### 8.2. Fallback-поведение

| Сценарий | Fallback-действие |
|----------|-------------------|
| **Ошибка глубокой ссылки** | Открыть главный экран |
| **Нет данных для отображения** | Показать уведомление без действия |
| **Неизвестный тип push** | Показать уведомление, не выполнять действие |

## 9. Сохранение истории уведомлений

### 9.1. Локальное хранение

| Элемент | Описание |
|---------|----------|
| **Максимум записей** | 50 последних уведомлений |
| **Сортировка** | По убыванию даты (свежие сверху) |
| **Хранение** | Persistent storage (Room / SQLite) |
| **Очистка** | При выходе из аккаунта |

### 9.2. Структура записи

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | string | Уникальный ID уведомления |
| `type` | string | Тип уведомления |
| `title` | string | Заголовок |
| `body` | string | Текст |
| `data` | object | Дополнительные данные |
| `receivedAt` | timestamp | Время получения |
| `read` | boolean | Прочитано ли |
| `action` | string | Действие по нажатию (глубокая ссылка) |

## 10. Критерии приёмки

1. Все типы push-уведомлений корректно отображаются
2. В активном состоянии показывается in-app баннер
3. В фоновом состоянии показывается системное уведомление
4. В закрытом состоянии показывается системное уведомление
5. Нажатие на уведомление открывает правильный экран
6. Глубокие ссылки работают корректно во всех состояниях
7. UI обновляется при получении уведомлений об изменениях
8. Кеш инвалидируется при получении уведомлений об изменениях
9. История уведомлений сохраняется локально
10. Ошибки обрабатываются корректно без крашей
