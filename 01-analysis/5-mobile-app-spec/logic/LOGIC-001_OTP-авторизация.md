# OTP-авторизация

> **ID:** LOGIC-001
> **Тип:** Бизнес-логика / Сценарий
> **Приоритет:** P0
> **Связанные требования:** FR-001, FR-002, FR-003, FR-004, FR-005; UC-001

---

## 1. Назначение

Описание процесса OTP-авторизации (One-Time Password) для клиентского приложения. Охватывает регистрацию, вход и верификацию по телефону и email, а также интеграцию с Telegram OAuth.

**Сценарии:**
- Регистрация нового клиента по телефону с подтверждением по SMS
- Регистрация нового клиента по email с подтверждением по почте
- Регистрация нового клиента через Telegram OAuth
- Вход существующего клиента по телефону/email с OTP
- Вход существующего клиента через Telegram OAuth

## 3. Состояния экрана регистрации/входа

| Состояние | Описание | UI-поведение |
|-----------|----------|--------------|
| **Начальное** | Поля для ввода данных | Все поля пусты, кнопка неактивна |
| **Заполнение** | Пользователь вводит данные | Кнопка становится активной (при валидации) |
| **Валидация** | Проверка данных на клиенте | Подсветка полей: зелёная (ок) / красная (ошибка) |
| **Отправка кода** | Запрос на отправку OTP | Кнопка → спиннер, поля заблокированы |
| **Ожидание кода** | Экран подтверждения OTP | Отображение 6 полей, таймер 30 сек |
| **Проверка кода** | Валидация OTP | Кнопка → спиннер, поля заблокированы |
| **Успех** | Код подтверждён | Плавный переход на главный экран |
| **Ошибка** | Ошибка валидации/сервера | Отображение сообщения, повторная попытка |

## 6. Обработка ошибок

| Сценарий | Действие клиента | Сообщение пользователю |
|----------|------------------|----------------------|
| Нет интернета | Проверка соединения | "Нет подключения к интернету. Проверьте соединение" |
| Сервер недоступен | Retry с экспоненциальной задержкой | "Сервис временно недоступен. Попробуйте позже" |
| Таймаут запроса | Retry (макс 3 раза) | "Что-то пошло не так. Попробуйте позже" |
| Ошибка 400 (неверный код) | Очистка полей OTP | "Неверный код. Попробуйте ещё раз" |
| Ошибка 409 (телефон/email занят) | Ссылка "Войти в аккаунт" | "Этот номер/email уже используется. Войдите в аккаунт" |
| Ошибка 422 (возраст < 6 лет) | — | "Регистрация доступна с 6 лет" |
| Истёкший код (410) | Активация повторной отправки | "Код устарел. Запросите новый" |

## 7. Таймер повторной отправки

| Состояние | Отображение | Действие |
|-----------|-------------|----------|
| **Исходное** | "Отправить код повторно" | Активна |
| **После отправки** | "Отправить код повторно (через 30с)" | Неактивна, таймер обратного отсчёта |
| **Таймер истёк** | "Отправить код повторно" | Активна |

**Количество попыток:** Максимум 5 попыток в течение 15 минут. При превышении — блокировка на 30 минут.

---

## 8. Критерии приёмки

1. Пользователь может зарегистрироваться по телефону с подтверждением по SMS
2. Пользователь может зарегистрироваться по email с подтверждением по почте
3. Пользователь может зарегистрироваться через Telegram OAuth (автоматическое создание аккаунта)
4. Валидация возраста (≥ 6 лет) блокирует регистрацию
5. Валидация уникальности телефона/email блокирует дубликаты
6. После успешной верификации клиент получает access и refresh токены
7. Токены сохраняются в безопасном хранилище
8. Неверный OTP-код показывает ошибку и позволяет повторить
9. Истёкший OTP-код позволяет запросить новый
10. При ошибках сети/сервера показываются понятные сообщения на русском языке
11. Таймер повторной отправки работает корректно
