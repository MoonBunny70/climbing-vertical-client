# Расчёт цены брони

> **ID:** LOGIC-003
> **Тип:** Бизнес-логика / Алгоритм
> **Приоритет:** P0
> **Связанные требования:** FR-026, FR-090; UC-004; US-021, US-090

---

## 1. Назначение

Логика расчёта итоговой стоимости брони на основе выбранных клиентом опций. Обеспечивает прозрачность ценообразования при записи на тренировку и обновляется в реальном времени при изменении выбора проката.

**Сценарии:**
- Расчёт цены тренировки с учётом проката снаряжения
- Отображение цены тренировки в карточке слота
- Отображение цены проката при выборе опций
- Обновление итоговой суммы в реальном времени при изменении выбора

---

## 2. Исходные данные

### 2.1. Данные слота

| Поле | Тип | Описание |
|------|-----|----------|
| `price` | decimal | Базовая цена тренировки |
| `equipmentRentalOptions` | array | Список доступных опций проката с ценами |

### 2.2. Опции проката

| Поле | Тип | Описание |
|------|-----|----------|
| `option` | string | Тип проката (SHOES / HARNESS) |
| `pricePerUnit` | decimal | Цена за единицу проката |

## 4. Формула расчёта

### 4.1. Базовая формула

totalPrice = basePrice + Σ(rentalOption.pricePerUnit)


Где:
- `basePrice` — цена тренировки из API
- `rentalOption.pricePerUnit` — цена за единицу проката для каждой выбранной опции

### 4.2. Примеры расчёта

#### Пример 1: Без проката (со своим снаряжением)

| Компонент | Значение |
|-----------|----------|
| Цена тренировки | 1200 ₽ |
| Выбранный прокат | Нет |
| **Итого** | **1200 ₽** |

#### Пример 2: Скальники (прокат)

| Компонент | Значение |
|-----------|----------|
| Цена тренировки | 1200 ₽ |
| Скальники | +200 ₽ |
| **Итого** | **1400 ₽** |

#### Пример 3: Скальники + страховочная система (прокат)

| Компонент | Значение |
|-----------|----------|
| Цена тренировки | 1200 ₽ |
| Скальники | +200 ₽ |
| Страховочная система | +300 ₽ |
| **Итого** | **1700 ₽** |

## 5. Формула отображения

### 5.1. Детализированное отображение

Итого: {basePrice} ₽ + {rentalTotal} ₽ = {totalPrice} ₽


### 5.2. Примеры отображения

| Прокат | Отображение |
|--------|-------------|
| Без проката | "Итого: 1200 ₽" |
| Скальники | "Итого: 1200 ₽ + 200 ₽ = 1400 ₽" |
| Страховочная система | "Итого: 1200 ₽ + 300 ₽ = 1500 ₽" |
| Скальники + страховочная система | "Итого: 1200 ₽ + 500 ₽ = 1700 ₽" |

### 5.3. Краткое отображение (для карточек)

💰 {totalPrice} ₽


---

## 6. Реактивное обновление

### 6.1. События, вызывающие перерасчёт

| Событие | Действие |
|---------|----------|
| Выбор опции проката | Добавление цены опции к итогу |
| Снятие опции проката | Вычитание цены опции из итога |
| Переключение на "Со своим снаряжением" | Сброс всех опций проката, totalPrice = basePrice |
| Переключение на "Взять в прокат" | Отображение доступных опций, расчёт с учётом выбранных |

### 6.2. Время обновления

- **Мгновенное** — при каждом изменении выбора клиента
- **Без задержки** — не требуется нажатия кнопки для обновления итога
- **Оптимизация** — не отправлять запросы на сервер при каждом изменении, только локальный расчёт

## 7. Валидация цены

### 7.1. Клиентская валидация

| Проверка | Описание |
|----------|----------|
| Цена тренировки > 0 | Базовая цена не может быть отрицательной или нулевой |
| Цена проката ≥ 0 | Цена проката не может быть отрицательной |
| Итоговая цена ≥ цены тренировки | Итог не может быть меньше базовой цены |

### 7.2. Серверная валидация

| Проверка | Описание |
|----------|----------|
| Актуальность цены | Сервер может пересчитать цену при создании брони |
| Доступность опций | Выбранные опции должны быть доступны на сервере |
| Корректность итога | Итоговая цена должна совпадать с расчётом сервера |

## 10. Критерии приёмки

1. Базовая цена тренировки отображается корректно из данных API
2. При выборе опции проката цена добавляется к итогу мгновенно
3. При снятии опции проката цена вычитается из итога мгновенно
4. При переключении на "Со своим снаряжением" все опции сбрасываются
5. Итоговая сумма отображается в формате "{basePrice} + {rentalTotal} = {total} ₽"
6. В карточке слота отображается только итоговая цена
7. Форматирование цены корректно (разделители тысяч, символ рубля)
8. Расчёт выполняется локально на клиенте без запросов к серверу
