# Отмена — правило 2 часов (порог n минут)

> **ID:** LOGIC-004
> **Тип:** Бизнес-правило / Алгоритм
> **Приоритет:** P0
> **Связанные требования:** FR-031, FR-032, FR-033, FR-034, FR-035; UC-005, UC-006; US-031, US-032

---

## 1. Назначение

Логика применения правила отмены и изменения проката с порогом `n` минут до начала тренировки. Значение `n` получается из бэкенда и настраивается в админке (по умолчанию — 120 минут = 2 часа).

**Сценарии:**
- Определение, освобождается ли место при отмене
- Определение, доступно ли изменение проката
- Отображение информации о пороге отмены в интерфейсе
- Расчёт времени, оставшегося до порога

## 2. Исходные данные

|------|-----|----------|
| `slot.startTime` | datetime | Время начала тренировки |
| `currentTime` | datetime | Текущее время (серверное или локальное) |

| Поле | Тип | Описание |
|------|-----|----------|
| `CANCELLATION_THRESHOLD_MINUTES` | integer | Порог `n` в минутах (по умолчанию 120) |

### 2.3. API эндпоинты для получения параметра

| Эндпоинт | Метод | Описание |
|----------|-------|----------|
| `/system/parameters` | GET | Получение системных параметров (кэшируется на клиенте) |

## 4. Матрица принятия решений

### 4.1. Отмена записи

| Условие | minutesUntilStart >= threshold | minutesUntilStart < threshold |
|---------|--------------------------------|-------------------------------|
| **Отмена доступна** | ✅ Да | ✅ Да |
| **Место освобождается** | ✅ Да | ❌ Нет |
| **UI предупреждение** | "Место освободится" | "Место НЕ освободится" |

### 4.2. Изменение проката

| Условие | minutesUntilStart >= threshold | minutesUntilStart < threshold |
|---------|--------------------------------|-------------------------------|
| **Изменение доступно** | ✅ Да | ❌ Нет |
| **UI предупреждение** | — | "Изменение проката недоступно" |

### 4.3. Общая матрица

| minutesUntilStart >= threshold | Отмена | Освобождение места | Изменение проката |
|--------------------------------|--------|-------------------|-------------------|
| Да (до порога) | ✅ | ✅ | ✅ |
| Нет (после порога) | ✅ | ❌ | ❌ |

## 5. Расчёт времени до порога

### 5.1. Формула

timeUntilThreshold = threshold - minutesUntilStart


Где:
- `threshold` — порог в минутах (например, 120)
- `minutesUntilStart` — минут до начала тренировки

### 5.2. Интерпретация результата

| timeUntilThreshold | Интерпретация |
|--------------------|---------------|
| > 0 | До порога осталось X минут |
| = 0 | Порог наступил сейчас |
| < 0 | Порог уже пройден (X минут назад) |

### 5.3. Форматирование для UI

```kotlin
fun formatTimeUntilThreshold(minutes: Int): String {
    return when {
        minutes > 0 -> "До порога отмены осталось ${formatTime(minutes)}"
        minutes == 0 -> "Порог отмены наступил"
        else -> "Порог отмены пройден ${formatTime(-minutes)} назад"
    }
}

fun formatTime(minutes: Int): String {
    val hours = minutes / 60
    val mins = minutes % 60
    return when {
        hours > 0 && mins > 0 -> "$hours ч $mins мин"
        hours > 0 -> "$hours ч"
        else -> "$mins мин"
    }
}
```
