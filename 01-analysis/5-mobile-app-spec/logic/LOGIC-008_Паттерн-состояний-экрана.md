# Паттерн состояний экрана

> **ID:** LOGIC-008
> **Тип:** Архитектурный паттерн
> **Приоритет:** P0
> **Связанные требования:** NFR-009; Все экраны

---

## 1. Назначение

Единый паттерн управления состояниями экранов, обеспечивающий консистентное отображение загрузки, данных, пустых состояний и ошибок. Паттерн применяется ко всем экранам приложения с динамическими данными.

**Сценарии:**
- Отображение загрузки при получении данных
- Отображение данных после успешной загрузки
- Отображение empty state при отсутствии данных
- Отображение ошибки при сбое загрузки
- Восстановление после ошибки

## 3. Определение состояний

### 3.1. Базовый набор состояний

| Состояние | Описание | Применение |
|-----------|----------|------------|
| **LOADING** | Данные загружаются | Показ Skeleton Screen |
| **CONTENT** | Данные успешно получены | Отображение контента |
| **EMPTY** | Данные отсутствуют | Отображение empty state |
| **ERROR** | Ошибка загрузки данных | Отображение ошибки |

### 3.2. Дополнительные состояния (опционально)

| Состояние | Описание | Применение |
|-----------|----------|------------|
| **REFRESHING** | Обновление данных (pull-to-refresh) | Показ индикатора обновления поверх контента |
| **LOADING_MORE** | Загрузка дополнительных данных | Показ индикатора в нижней части списка |
| **SUCCESS** | Действие выполнено успешно | Показ тост-сообщения или модалки |

## 5. Переходы между состояниями

### 5.1. Матрица переходов

| Из → В | LOADING | CONTENT | EMPTY | ERROR |
|--------|---------|---------|-------|-------|
| **LOADING** | — | ✅ | ✅ | ✅ |
| **CONTENT** | ❌ | — | ❌ | ❌ |
| **EMPTY** | ❌ | ✅ | — | ✅ |
| **ERROR** | ✅ | ✅ | ✅ | — |

### 5.2. Анимации переходов

| Переход | Анимация | Длительность |
|---------|----------|--------------|
| LOADING → CONTENT | Fade-in контента | 0.3 сек |
| LOADING → EMPTY | Fade-in empty state | 0.3 сек |
| LOADING → ERROR | Fade-in ошибки | 0.3 сек |
| ERROR → LOADING | Fade-out ошибки, затем Skeleton | 0.2 сек |
| EMPTY → LOADING | Fade-out empty, затем Skeleton | 0.2 сек |

## 6. Обработка действий

### 6.1. Кнопка "Обновить" / "Повторить"

| Действие | Реакция |
|----------|---------|
| Нажатие на "Обновить" | Переход в состояние LOADING, повторный запрос данных |
| Успешный запрос | Переход в состояние CONTENT или EMPTY |
| Ошибка запроса | Остаётся в состоянии ERROR |

### 6.2. Pull-to-refresh

| Действие | Реакция |
|----------|---------|
| Pull-to-refresh в состоянии CONTENT | Переход в состояние REFRESHING, запрос данных |
| Успешный запрос | Возврат в состояние CONTENT с обновлёнными данными |
| Ошибка запроса | Возврат в состояние CONTENT с уведомлением об ошибке |

## 7. Иерархия состояний

### 7.1. Глобальные состояния

| Уровень | Состояние | Описание |
|---------|-----------|----------|
| **Глобальный** | OFFLINE | Нет интернет-соединения (баннер сверху) |
| **Глобальный** | UNAUTHORIZED | Токен истёк (переход на экран входа) |
| **Экранный** | LOADING / CONTENT / EMPTY / ERROR | Состояние конкретного экрана |

### 7.2. Вложенные состояния

Экран
├── LOADING (Skeleton)
├── CONTENT
│ ├── Данные отображены
│ └── REFRESHING (индикатор обновления)
├── EMPTY
│ ├── Иконка + Заголовок + Подзаголовок
│ └── Кнопка действия (опционально)
└── ERROR
├── Иконка + Заголовок + Подзаголовок
└── Кнопка "Обновить"


---

## 8. Критерии приёмки

1. Все экраны с динамическими данными используют единый паттерн состояний
2. Состояние LOADING отображается при первой загрузке данных
3. Skeleton Screen анимирован (пульсация)
4. Состояние CONTENT отображает данные после успешной загрузки
5. Состояние EMPTY отображается при отсутствии данных с понятным сообщением
6. Состояние ERROR отображается при ошибке загрузки с кнопкой "Обновить"
7. Все переходы между состояниями плавные и анимированные
8. Pull-to-refresh работает в состоянии CONTENT
9. При ошибке запроса пользователь может повторить попытку
10. Все сообщения для пользователя понятны и на русском языке
