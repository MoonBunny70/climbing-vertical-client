# Use Cases (сценарии использования)

> Источник: [`../1-elicitation/domain-description.md`](../1-elicitation/domain-description.md)  
> Трассировка: [`functional-requirements.md`](functional-requirements.md), [`user-stories.md`](user-stories.md)

**Акторы:** Клиент, Система (клиентское приложение + API), Бэкенд (black box), Скалодром (инициирует изменения через существующую инфраструктуру).

## UC-001 — Регистрация клиента

| Поле | Значение |
|------|----------|
| **ID** | UC-001 |
| **Название** | Регистрация клиента |
| **Актор** | Клиент |
| **Приоритет** | P0 |
| **FR** | FR-001, FR-002, FR-003, FR-004, FR-005 |
| **US** | US-001, US-002, US-003, US-004 |
| **Источник** | §4.1 |

**Предусловия:**
- Клиент не авторизован
- Выбран способ регистрации: телефон, Telegram или email

**Основной поток:**
1. Клиент инициирует регистрацию выбранным способом.
2. Система запрашивает необходимые данные и проверяет возраст (≥ 6 лет).
3. Система создаёт **один аккаунт** на клиента через API.
4. Клиент авторизован и получает доступ к функциям приложения.

**Альтернативные потоки:**
- **1a.** Возраст &lt; 6 лет → система отклоняет регистрацию с сообщением об ограничении.
- **2a.** Ошибка API → регистрация не завершена, клиенту показана ошибка.

**Постусловия:**
- Создан аккаунт клиента; клиент авторизован.

## UC-003 — Просмотр расписания

| Поле | Значение |
|------|----------|
| **ID** | UC-003 |
| **Название** | Просмотр расписания тренировок |
| **Актор** | Клиент |
| **Приоритет** | P0 |
| **FR** | FR-010, FR-011, FR-012, FR-013, FR-014, FR-015 |
| **US** | US-010, US-011 |
| **Источник** | §4.2, §5.6, R-027 |

**Предусловия:** Клиент авторизован (или приложение допускает просмотр — по реализации; запись требует авторизации).

**Основной поток:**
1. Клиент открывает экран расписания.
2. Система запрашивает слоты у API на горизонт **≤ 7 дней**.
3. Система отображает карточки слотов: время, 1,5 ч, формат (1 из 2), инструктор, зона, свободные места, цена.
4. Слоты, отменённые скалодромом, отображаются как недоступные для записи.

**Альтернативные потоки:**
- **2a.** Слотов нет → empty state «Пока нет доступных тренировок».

**Постусловия:** Клиент видит актуальное расписание в рамках 7-дневного горизонта.

## UC-004 — Запись на тренировку

| Поле | Значение |
|------|----------|
| **ID** | UC-004 |
| **Название** | Запись на тренировку |
| **Актор** | Клиент |
| **Приоритет** | P0 |
| **FR** | FR-020–FR-027, FR-090 |
| **US** | US-020, US-021, US-022, US-090 |
| **Источник** | §4.3, §5.1, §7, R-004 |

**Предусловия:**
- Клиент авторизован
- У клиента **нет активного долга**
- Выбран доступный слот

**Основной поток:**
1. Клиент открывает детали слота и видит цену тренировки.
2. Клиент выбирает опции проката из перечня API; система показывает цену проката.
3. При нехватке прокатного снаряжения система показывает **предупреждение**.
4. Клиент подтверждает запись.
5. Система отправляет запрос в API; при успехе бронь **мгновенно** получает статус «Подтверждена».

**Альтернативные потоки:**
- **0a.** Активный долг → запись заблокирована (UC-011).
- **5a.** API отклонил (нет мест) → бронь не создана, клиенту показана ошибка.
- **3a.** Клиент продолжает запись несмотря на предупреждение о прокате (если API разрешает).

**Постусловия:** Создана запись со статусом «Подтверждена» или запись отклонена.

## UC-005 — Отмена записи клиентом

| Поле | Значение |
|------|----------|
| **ID** | UC-005 |
| **Название** | Отмена записи клиентом |
| **Актор** | Клиент |
| **Приоритет** | P0 |
| **FR** | FR-031, FR-032, FR-035 |
| **US** | US-031 |
| **Источник** | §5.2, §7 |

**Предусловия:**
- Клиент авторизован
- Есть активная запись со статусом «Подтверждена»
- Система знает значение **`n` минут** из API

**Основной поток:**
1. Клиент открывает свою запись и инициирует отмену.
2. Система определяет, осталось ли до начала **более `n` минут**.
3. **3a.** Если да → отмена выполняется, статус «Отменена клиентом», **место освобождается**.
4. **3b.** Если нет → отмена выполняется, статус «Отменена клиентом», место **не освобождается**.

**Постусловия:** Запись отменена клиентом; правило освобождения места применено.

## UC-008 — Отмена тренировки скалодромом

| Поле | Значение |
|------|----------|
| **ID** | UC-008 |
| **Название** | Обработка отмены тренировки скалодромом |
| **Актор** | Скалодром (инициатор), Клиент (получатель), Система |
| **Приоритет** | P0 |
| **FR** | FR-040, FR-041, FR-081, FR-082 |
| **US** | US-040, US-081 |
| **Источник** | §5.3, §6.1, R-008 |

**Предусловия:**
- У клиента есть запись на слот
- Скалодром отменил слот через существующую инфраструктуру

**Основной поток:**
1. Бэкенд передаёт изменение статуса в API.
2. Система обновляет запись: статус **«Отменена скалодромом»**, отображается **причина**.
3. Система отправляет клиенту **push** с кнопкой **«Записаться на другой слот»**.
4. При наличии email — дублирование на почту (UC-013).
5. Слот недоступен для повторной записи.

**Постусловия:** Бронь сохранена с финальным статусом; клиент уведомлён.

## UC-013 — Настройка и доставка уведомлений

| Поле | Значение |
|------|----------|
| **ID** | UC-013 |
| **Название** | Настройка и доставка уведомлений |
| **Актор** | Клиент, Система |
| **Приоритет** | P0 |
| **FR** | FR-080, FR-081–FR-086, FR-087 |
| **US** | US-080–US-084 |
| **Источник** | §6.1, §6.2, §7 |

**Предусловия:** Клиент авторизован; для push — разрешения ОС; для email — email в профиле.

**Основной поток (настройка напоминания):**
1. Клиент задаёт время push-напоминания о предстоящей тренировке.
2. Система сохраняет настройку.

**Основной поток (доставка событий):**
1. Наступает событие: напоминание о тренировке / отмена скалодромом / смена времени / смена инструктора / напоминание об оценке.
2. Система отправляет push клиенту.
3. Если email указан — дублирование на email.

**Альтернативные потоки:**
- **2a.** Отмена скалодромом → push содержит действие «Записаться на другой слот».

**Постусловия:** Клиент получил уведомление по каналам согласно домену.

## UC-014 — Оценка инструктора

| Поле | Значение |
|------|----------|
| **ID** | UC-014 |
| **Название** | Оценка инструктора |
| **Актор** | Клиент |
| **Приоритет** | P1 |
| **FR** | FR-060–FR-064, FR-085 |
| **US** | US-060, US-061, US-062, US-083 |
| **Источник** | §4.6, §6.1 |

**Предусловия:** Клиент авторизован; инструктор доступен для оценки.

**Основной поток:**
1. Клиент открывает форму оценки инструктора.
2. Клиент ставит **звёзды** (обязательно).
3. Клиент опционально добавляет **комментарий** от имени аккаунта.
4. Система сохраняет оценку через API.

**Альтернативные потоки:**
- **4a.** Клиент добавляет или дополняет **комментарий позже** → сохраняется в любой момент.
- **2a.** Попытка изменить **звёзды** после отправки → система отклоняет; сообщает о необходимости обратиться к администратору.

**Постусловия:** Оценка инструктора (звёзды ± комментарий) сохранена.

## UC-015 — Переход к записи из уведомления об отмене

| Поле | Значение |
|------|----------|
| **ID** | UC-015 |
| **Название** | Запись на другой слот из уведомления |
| **Актор** | Клиент |
| **Приоритет** | P0 |
| **FR** | FR-082, FR-041 |
| **US** | US-081 |
| **Источник** | §5.3 п.3, §6.1 |

**Предусловия:** Клиент получил push об отмене скалодромом.

**Основной поток:**
1. Клиент нажимает **«Записаться на другой слот»** в уведомлении.
2. Система открывает расписание (UC-003).
3. Клиент выбирает другой слот и записывается (UC-004), если нет долга и есть места.

**Альтернативные потоки:**
- **3a.** Активный долг → UC-011.
- **3a.** Клиент пытается выбрать **отменённый скалодромом** слот → запись запрещена.

**Постусловия:** Клиент перенаправлен к новой записи или операция отклонена по правилам домена.
