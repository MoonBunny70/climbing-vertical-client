# API Sequence-диаграммы — Клиентское приложение «Вертикаль»

> **Назначение:** Описание ключевых API-взаимодействий клиентского приложения с бэкендом. 
> Бэкенд рассматривается как **black-box** — клиент полагается на контракт API и корректно обрабатывает ответы.
>
> **Принципы:**
> - Все запросы требуют авторизации (кроме регистрации/входа)
> - Бэкенд — источник истины, гарантирует атомарность операций
> - Клиент обрабатывает все коды ответов (2xx, 4xx, 5xx)
>
> **Источники:** `domain-description.md`, `functional-requirements.md`, `use-cases.md`

---

## Легенда последовательностей

| Обозначение | Смысл |
|-------------|-------|
| **Клиент** | Мобильное приложение (Android) |
| **API** | Backend API (black-box) |
| **→** | Синхронный запрос/ответ |
| **--→** | Асинхронное уведомление (push) |
| **alt** | Альтернативные потоки (условия) |

## Сводка кодов ответов

| Код | Описание | Обработка на клиенте |
|-----|----------|----------------------|
| **200 OK** | Успешный запрос | Отображение данных |
| **201 Created** | Запись создана | Показ модалки успеха |
| **400 Bad Request** | Неверные данные | Показ ошибки с указанием поля |
| **401 Unauthorized** | Невалидный токен | Обновление токена / переход на вход |
| **403 Forbidden** | Доступ запрещён (долг, нет прав) | Показ баннера долга / ошибки |
| **404 Not Found** | Сущность не найдена | Показ ошибки "Не найдено" |
| **409 Conflict** | Конфликт (нет мест, дубликат) | Показ ошибки "Мест нет" |
| **410 Gone** | Слот отменён | Показ ошибки с причиной |
| **422 Unprocessable** | Валидационная ошибка | Показ ошибки валидации |
| **500 Internal Error** | Ошибка сервера | Retry, показ "Что-то пошло не так" |
| **503 Service Unavailable** | Сервер недоступен | Retry с задержкой |
