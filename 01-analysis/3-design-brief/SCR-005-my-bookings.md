# Экран моих записей

> **ID:** SCR-005
> **Приоритет:** P0
> **Связанные требования:** FR-030; BR-12; US-030; UC-007

---

## 1. Назначение экрана

Отображение всех записей клиента на тренировки с разделением по статусам. Позволяет клиенту видеть историю и управлять активными записями.

**Сценарии:**
- Клиент видит все свои записи (активные и завершённые)
- Клиент видит статус каждой записи
- Клиент переходит к деталям конкретной записи для управления (отмена, просмотр)
- Клиент видит записи, отменённые скалодромом, с указанием причины

## 3. Компоненты экрана

### 3.1. Tab-фильтры статусов

┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐
│ Активные│ │ Предстоящие│ │ Завершённые│ │ Отменённые│
│ (3) │ │ (2) │ │ (5) │ │ (2) │
└──────────┘ └──────────┘ └──────────┘ └──────────┘


| Элемент | Описание |
|---------|----------|
| **Активные** | Записи со статусом «Подтверждена» на будущие даты (включая сегодня) |
| **Предстоящие** | Записи со статусом «Ожидание» (если есть) или будущие записи |
| **Завершённые** | Записи со статусом «Завершена» (прошедшие тренировки) |
| **Отменённые** | Записи со статусом «Отменена клиентом» и «Отменена скалодромом» |

**Визуальное состояние:**
- **Активный таб:** Оранжевый фон, белый текст, скругление 20 pt
- **Неактивный таб:** Прозрачный фон, тёмно-серый текст
- **Бейдж с количеством:** Маленький круг с числом рядом с названием

## 4. Цветовая схема статусов

| Статус | Цвет | Индикатор | Применение |
|--------|------|-----------|------------|
| **Подтверждена** | Зелёный `#22C55E` | 🟢 | Активные будущие записи |
| **Ожидание** | Жёлтый `#F59E0B` | 🟡 | Если есть промежуточный статус |
| **Отменена клиентом** | Серый `#9CA3AF` | ⚪ | Отменённые записи |
| **Отменена скалодромом** | Серый `#9CA3AF` | ⚪ | Отменены организатором |
| **Завершена** | Синий `#3B82F6` | 🔵 | Прошедшие тренировки |
| **No-show / долг** | Красный `#EF4444` | 🔴 | Неявка без оплаты |

## 5. Состояния экрана

| Состояние | Описание | Визуальное поведение |
|-----------|----------|---------------------|
| **Загрузка** | Данные загружаются | Skeleton screen для карточек |
| **С данными** | Есть записи | Полный список с группировкой по статусам |
| **Нет записей в выбранной категории** | В категории нет записей | Empty state для категории: «Здесь пока ничего нет» |
| **Нет записей вообще** | Нет ни одной записи | Полный empty state экрана |
| **Ошибка** | Ошибка загрузки | Сообщение об ошибке + кнопка «Обновить» |

---

## 6. Взаимодействия

| Действие | Реакция |
|----------|---------|
| **Нажатие на таб** | Переключение между категориями; обновление списка |
| **Нажатие на карточку** | Переход на экран деталей записи (SCR-006) |
| **Нажатие «Подробнее»** | Переход на экран деталей записи (SCR-006) |
| **Нажатие «Оценить инструктора»** | Переход на экран оценки инструктора (SCR-009) |
| **Pull-to-refresh** | Обновление списка записей |
| **Скролл** | Плавная прокрутка списка |

## 7. Обработка ошибок

| Сценарий | Сообщение пользователю | Действие |
|----------|----------------------|----------|
| Ошибка загрузки | «Не удалось загрузить ваши записи. Проверьте интернет и попробуйте снова» | Кнопка «Обновить» |
| Нет записей в категории | «Здесь пока ничего нет» | — |
| Нет записей вообще | «У вас пока нет записей. Запишитесь на тренировку прямо сейчас!» | Кнопка «Перейти к расписанию» |
| Нет интернета | «Нет подключения к интернету. Показываем сохранённые данные» | Баннер сверху |

## 8. Навигация

Главный экран → Мои записи (по кнопке «Мои записи»)
Мои записи → Детали записи (по карточке / кнопке «Подробнее»)
Мои записи → Оценка инструктора (по кнопке «Оценить инструктора»)
Мои записи ← Главный экран (кнопка «Назад»)

---

## 9. Критерии приёмки

1. Отображаются все записи клиента с разделением по статусам
2. Tab-фильтры работают: Активные / Предстоящие / Завершённые / Отменённые
3. У каждого таба отображается количество записей
4. Каждая карточка содержит: дату/время, формат, инструктора, зону, статус, прокат
5. Для статуса «Отменена скалодромом» отображается причина
6. Для статуса «Завершена» доступна кнопка «Оценить инструктора» (если оценка не оставлена)
7. Для статуса «No-show / долг» отображается сумма долга
8. При отсутствии записей показывается empty state с переходом к расписанию
9. Pull-to-refresh обновляет список
10. Поддержка тёмной темы (опционально)
