# Экран настроек уведомлений

> **ID:** SCR-010
> **Приоритет:** P1
> **Связанные требования:** FR-080, FR-087; BR-9; US-080, US-084; UC-013

---

## 1. Назначение экрана

Настройка времени push-напоминаний о предстоящих тренировках и просмотр истории уведомлений.

**Сценарии:**
- Клиент настраивает время напоминания о тренировке
- Клиент включает/отключает push-уведомления (через настройки ОС)
- Клиент просматривает историю уведомлений
- Клиент видит дублирование на email

## 3. Компоненты экрана

### 3.1. Настройка времени напоминания

| Элемент | Описание |
|---------|----------|
| **Заголовок** | «🔔 Напоминание о тренировке» |
| **Выбор времени** | Dropdown с вариантами времени |
| **Информация** | «Напоминание придёт в указанное время до тренировки» |

### 3.2. Email-дублирование

| Элемент | Описание |
|---------|----------|
| **Заголовок** | «✉️ Email-дублирование» |
| **Toggle** | Включить/отключить отправку на email |
| **Email** | Отображение email клиента (если указан) |

### 3.3. История уведомлений

| Элемент | Описание |
|---------|----------|
| **Заголовок** | «📅 История уведомлений» |
| **Строка уведомления** | Дата/время + иконка типа + текст уведомления |

**Типы уведомлений:**
| Иконка | Тип | Описание |
|--------|-----|----------|
| ⏰ | Напоминание | Напоминание о предстоящей тренировке |
| ❌ | Отмена | Отмена скалодромом |
| 📅 | Изменение | Изменение времени или инструктора |
| 🔔 | Другое | Напоминание об оценке и т.п. |

## 4. Состояния экрана

| Состояние | Описание | Визуальное поведение |
|-----------|----------|---------------------|
| **Загрузка** | Данные загружаются | Skeleton screen |
| **С данными** | Настройки загружены | Полный интерфейс |
| **Сохранение** | Настройки сохраняются | Индикатор загрузки на кнопке/переключателе |
| **Ошибка** | Ошибка сохранения | Snackbar с сообщением об ошибке |

## 5. Взаимодействия

| Действие | Реакция |
|----------|---------|
| **Изменение времени напоминания** | Сохранение настройки через API |
| **Переключение email-дублирования** | Сохранение настройки через API |
| **Нажатие на уведомление в истории** | (Опционально) Переход к связанной записи/слоту |

## 6. Обработка ошибок

| Сценарий | Сообщение пользователю | Действие |
|----------|----------------------|----------|
| Ошибка сохранения настроек | «Не удалось сохранить настройки. Попробуйте позже» | Snackbar |
| Нет интернета | «Нет подключения к интернету. Проверьте соединение» | Snackbar |

---

## 7. Навигация

Профиль → Настройки уведомлений (по пункту меню)
Настройки уведомлений ← Профиль (кнопка «Назад»)


---

## 8. Критерии приёмки

1. Клиент может выбрать время напоминания о тренировке из списка
2. Выбранное время сохраняется и применяется для всех будущих тренировок
3. Клиент может включить/отключить email-дублирование уведомлений
4. Отображается email клиента (если указан)
5. Отображается история последних уведомлений с датой и типом
6. Настройки сохраняются через API
7. Поддержка тёмной темы (опционально)
