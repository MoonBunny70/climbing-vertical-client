# Экран входа / регистрации

> **ID:** SCR-001
> **Приоритет:** P0
> **Связанные требования:** FR-001, FR-002, FR-003, FR-004, FR-005; BR-2; US-001, US-002, US-003; UC-001

---

## 1. Назначение экрана

Авторизация и регистрация клиента. Первая точка контакта с приложением. Пользователь может войти в существующий аккаунт или зарегистрироваться новым.

**Сценарии:**
- Новый клиент регистрируется по телефону → подтверждение SMS → переход на главный экран
- Новый клиент регистрируется через Telegram → OAuth → переход на главный экран
- Новый клиент регистрируется по email → подтверждение → переход на главный экран
- Существующий клиент входит в аккаунт → переход на главный экран

## 3. Состояния экрана

| Состояние | Визуальное поведение |
|-----------|---------------------|
| **Начальное** | Поля пусты, кнопка «Зарегистрироваться» неактивна (серая) |
| **Заполнение** | Поля валидны → кнопка активна (оранжевая) |
| **Валидация** | Подсветка поля: зелёная рамка (валидно) / красная рамка с сообщением (ошибка) |
| **Загрузка** | Кнопка показывает спиннер, текст меняется на «Загрузка...», поля заблокированы |
| **Успех** | Код подтверждён → плавный переход на главный экран |
| **Ошибка сервера** | Показывается сообщение с ошибкой, кнопка «Повторить» |
| **Ожидание кода** | Переход на экран подтверждения, таймер 30 секунд |

## 4. Компоненты экрана

### Поле ввода номера телефона

┌─────────────────────────────────────────────────────┐
│ 📱 Номер телефона │ ← Label (14pt)

│ │ +7 (999) 123-45-67 │ │ ← Input (16pt)
│ └─────────────────────────────────────────────┘ │

- **Маска:** `+7 (XXX) XXX-XX-XX`
- **Клавиатура:** Phone Pad
- **Валидация:** 11 цифр после +7

## 5. Обработка ошибок

| Сценарий | Сообщение пользователю | Визуальное состояние |
|----------|----------------------|---------------------|
| Неверный номер | «Введите корректный номер телефона» | Красная рамка поля |
| Возраст < 6 лет | «Регистрация доступна с 6 лет» | Красная рамка поля + сообщение под ним |
| Email уже зарегистрирован | «Этот email уже используется. Войдите в аккаунт» | Красная рамка + ссылка на вход |
| Телефон уже зарегистрирован | «Этот номер уже используется. Войдите в аккаунт» | Красная рамка + ссылка на вход |
| Неверный код | «Неверный код. Попробуйте ещё раз» | Красная рамка всех полей |
| Истёк срок кода | «Код устарел. Запросите новый» | Активация ссылки повторной отправки |
| Ошибка сервера | «Что-то пошло не так. Попробуйте позже» | Snackbar внизу экрана |
| Нет интернета | «Нет подключения к интернету. Проверьте соединение» | Snackbar внизу экрана |

## 6. Навигация

Вход → Регистрация (по ссылке «Нет аккаунта?»)
Регистрация → Вход (по ссылке «Уже есть аккаунт?»)
Регистрация → Подтверждение (после отправки SMS/email)
Подтверждение → Главный экран (после успешного подтверждения)
Вход → Главный экран (после успешного входа)


---

## 7. Критерии приёмки

1. Пользователь может зарегистрироваться по телефону с подтверждением по SMS
2. Пользователь может зарегистрироваться через Telegram OAuth
3. Пользователь может зарегистрироваться по email с подтверждением
4. Регистрация невозможна для пользователей младше 6 лет
5. После успешной регистрации пользователь автоматически авторизован и попадает на главный экран
6. Существующий пользователь может войти по телефону, Telegram или email
7. При повторном входе пользователь видит экран входа
8. Токен сессии сохраняется; повторный вход не требуется при перезапуске приложения
9. Все ошибки валидации отображаются понятными сообщениями на русском языке
10. Поддержка тёмной темы (опционально)
