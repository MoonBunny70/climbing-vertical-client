# Экран деталей слота / запись

> **ID:** SCR-004
> **Приоритет:** P0
> **Связанные требования:** FR-020, FR-021, FR-024, FR-025, FR-026, FR-027, FR-090; BR-4, BR-8; US-020, US-021, US-090; UC-004

---

## 1. Назначение экрана

Подробная информация о тренировочном слоте, выбор опций проката снаряжения и подтверждение записи.

**Сценарии:**
- Клиент видит детальную информацию о тренировке
- Клиент выбирает прокат снаряжения (скальники, страховочная система)
- Клиент видит цену тренировки и проката
- Клиент подтверждает запись (мгновенная)
- Клиент видит предупреждение о нехватке снаряжения

## 3. Компоненты экрана

### 3.1. Информационная карточка слота

| Элемент | Тип | Размер | Цвет | Описание |
|---------|-----|--------|------|----------|
| **Время** | Текст | 20pt, Bold | Тёмно-серый | «18:00 — 19:30» |
| **Формат** | Текст | 16pt, SemiBold | Тёмно-серый | «Болдеринг для новичков» |
| **Инструктор** | Текст | 16pt, Regular | Серый | «Инструктор: Анна С.» |
| **Зона** | Текст | 16pt, Regular | Серый | «📍 Стена А» |
| **Свободные места** | Текст | 16pt, Regular | Цвет статуса | «🟢 Свободно: 5 мест из 8» |
| **Цена тренировки** | Текст | 18pt, Bold | Оранжевый | «1200 ₽» |

### 3.2. Выбор проката снаряжения

#### Опция 1: «Со своим снаряжением»

┌─────────────────────────────────────────────────────────────┐
│ ● Со своим снаряжением │ ← Radio (активен)
│ (скальники и страховочная система) │ ← Подпись
└─────────────────────────────────────────────────────────────┘


- **Radio кнопка:** ● (активен) / ○ (неактивен)
- **Текст:** «Со своим снаряжением»
- **Подпись:** «(скальники и страховочная система)» — 12pt, серый

#### Опция 2: «Взять в прокат»

┌─────────────────────────────────────────────────────────────┐
│ ○ Взять в прокат │ ← Radio (неактивен)
│ ┌─────────────────────────────────────────────────────┐ │
│ │ ☑️ Скальники +200 ₽ │ │ ← Checkbox (активен)
│ │ ☑️ Страховочная система +300 ₽ │ │ ← Checkbox (активен)
│ └─────────────────────────────────────────────────────┘ │
│ ⚠️ Предупреждение: осталось только 2 пары скальников │ ← Warning
└─────────────────────────────────────────────────────────────┘

- **Radio кнопка:** ○ (неактивен) / ● (активен)
- **Checkbox:** ☑️ (активен) / ☐ (неактивен)
- **Чекбоксы** активны только когда выбрана опция «Взять в прокат»
- **Цена проката:** Отображается рядом с каждым пунктом

#### Предупреждение о нехватке снаряжения

⚠️ Предупреждение: осталось только 2 пары скальников в прокате


- **Иконка:** ⚠️
- **Текст:** 14pt, Regular, Жёлтый/Оранжевый цвет
- **Появляется,** если свободного снаряжения меньше, чем требуется

### 3.3. Блок «Итого»

┌─────────────────────────────────────────────────────────────┐
│ Итого: 1200 ₽ + 500 ₽ = 1700 ₽ │
└─────────────────────────────────────────────────────────────┘

- **Текст:** 16pt, Regular, Тёмно-серый
- **Итоговая сумма:** 18pt, Bold, Оранжевый
- **Отображение:** Только если выбран прокат

### 3.4. Баннер долга

┌─────────────────────────────────────────────────────────────┐
│ ⚠️ У вас есть долг за неявку. Обратитесь к │
│ администратору для погашения. │
└─────────────────────────────────────────────────────────────┘


- **Фон:** Красный `#EF4444` или жёлтый `#F59E0B`
- **Текст:** Белый, 14pt, Regular
- **Появляется,** если у клиента есть активный долг
- **Кнопка «Записаться»** блокируется

## 4. Состояния экрана

| Состояние | Описание | Визуальное поведение |
|-----------|----------|---------------------|
| **Загрузка** | Данные загружаются | Skeleton screen |
| **Готов к записи** | Данные загружены, пользователь выбирает прокат | Все элементы активны |
| **Выбор проката** | Пользователь выбирает опции | Обновление итоговой суммы в реальном времени |
| **Предупреждение** | Нехватка снаряжения | Жёлтый баннер под опциями проката |
| **Долг** | Активный долг | Баннер долга, кнопка заблокирована |
| **Запись в процессе** | Отправка запроса на сервер | Кнопка показывает спиннер, текст «Запись...» |
| **Успех** | Запись подтверждена | Модальное окно успеха |
| **Ошибка** | Ошибка записи | Snackbar с сообщением об ошибке |

---

## 5. Взаимодействия

| Действие | Реакция |
|----------|---------|
| **Выбор «Со своим снаряжением»** | Скрыть чекбоксы проката, сбросить прокат, обновить итоговую сумму |
| **Выбор «Взять в прокат»** | Показать чекбоксы, обновить итоговую сумму при выборе |
| **Выбор/снятие чекбокса** | Обновление итоговой суммы в реальном времени |
| **Нажатие «Записаться»** | Отправка запроса в API; кнопка переходит в состояние загрузки |
| **Успешная запись** | Показ модального окна успеха |
| **Ошибка записи** | Snackbar с ошибкой, кнопка возвращается в нормальное состояние |

---

## 6. Обработка ошибок

| Сценарий | Сообщение пользователю | Действие |
|----------|----------------------|----------|
| Не хватает снаряжения | «Предупреждение: осталось только X единиц снаряжения в прокате» | Жёлтый баннер; запись возможна |
| Не хватает снаряжения (блокировка) | «Недостаточно снаряжения в прокате для вашего выбора. Выберите другой вариант» | Кнопка «Записаться» неактивна (если API блокирует) |
| API отказ (нет мест) | «К сожалению, все места на эту тренировку уже заняты» | Кнопка возвращается в нормальное состояние |
| Ошибка сервера | «Что-то пошло не так. Попробуйте позже» | Snackbar, кнопка возвращается в нормальное состояние |
| Нет интернета | «Нет подключения к интернету. Проверьте соединение» | Snackbar |
| Активный долг | «У вас есть долг за неявку. Обратитесь к администратору для погашения» | Баннер долга, кнопка «Записаться» заблокирована |

## 7. Навигация

Расписание → Детали слота (по карточке / кнопке «Записаться»)
Детали слота ← Расписание (кнопка «Назад»)
Детали слота → Главный экран (после успешной записи)
Детали слота → Мои записи (после успешной записи, по кнопке в модалке)


---

## 8. Критерии приёмки

1. Отображается полная информация о слоте (время, формат, инструктор, зона, места, цена)
2. Пользователь может выбрать «Со своим снаряжением» или «Взять в прокат»
3. При выборе проката доступны чекбоксы «Скальники» и «Страховочная система» с ценами
4. Итоговая сумма обновляется в реальном времени при выборе опций
5. При нехватке прокатного снаряжения показывается предупреждение
6. При активном долге запись блокируется с показом баннера
7. При подтверждении записи кнопка переходит в состояние загрузки
8. После успешной записи показывается модальное окно с деталями
9. При ошибке API показывается понятное сообщение
10. Поддержка тёмной темы (опционально)
